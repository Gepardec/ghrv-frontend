General

This application is used to determine the unique/total views to the public GitHub repos of gepardec.

User interface

The UI is split into two parts:

  • Sidebar
  • Chart

Sidebar

Here you can set preferences like date range, repos to display, best repos (by total/unique views), et al.

Note the following:

  • The value inside best repos will always override the repo selection when refreshing the page.
  • The definition of "best" is calculated via the current view mode. If you select total views then the best n repos with the highest total views are displayed. For unique views this works vice versa.

Chart

This part displays the data corresponding to the preferences in the sidebar. You can select or deselect the repos in the chart legend without using the sidebar.

Caching

You may have noticed that this application caches some of your preferences set in the sidebar. View mode and best n are saved so you don't have to set them at every page refresh.

Note that date range is not cached, because we want to default yesterday until one month before as range.
